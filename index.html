Distributed SQLite for clusters

Fast, embedded, in-memory or persistent SQL database with RAFT consensus for high-availability IoT and Edge devices.

Dqlite extends SQLite across a cluster of hosts, with automatic failover and high availability to keep your application running.

It uses C-RAFT, an optimised RAFT implementation in C to gain high-performance transactional consensus and fault tolerance while preserving SQlite’s famous efficiency and footprint.

Enterprise-grade SQL database availability for the Edge and IoT

Modern edge computing is moving high-value applications to the edge of the network, where you often have far fewer guarantees about hardware availability and management. Appliances in the cabinet or in remote locations cannot be monitored and managed, or replaced very quickly, and often cannot have the same environmental controls. That makes the hardware less reliable, and increases the cost of human interventions.

Dqlite provides edge applications with an ultra-fast SQL database that is automatically replicated across multiple hosts, and guaranteed to provide an on-line backup for fast failover. SQLite is the world’s most widely used embedded SQL implementation. Dqlite adds robust consensus and failover semantics.

Why RAFT?

Consensus algorithms provide certainty of transaction replication so that it is always possible to guarantee data persistence in the event of the loss of a machine in the cluster. RAFT is widely considered the consensus algorithm of choice for efficiency and correctness of implementation. C-RAFT is a hand-tuned C implementation of RAFT that is fully asynchronous by design.
